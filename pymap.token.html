pymap.token¶

pymap.token.tokens: Plugin[type[pymap.interfaces.token.TokensInterface]] = Plugin('pymap.token')¶

Registers token plugins.

class pymap.token.AllTokens(*args, **kwargs)[source]¶

Uses tokens to support all registered token types.

For token creation, the default token plugin is used. For token parsing, each token plugin is tried until one succeeds.

get_login_token(identifier, authcid, key, *, authzid=None, location=None, expiration=None)[source]¶

Returns a new token string that encapsulates the provided login data, or None if tokens are not supported.

Parameters:

  • identifier (str) – Any string that can facilitate the lookup of key.

  • authcid (str) – The authentication identity of the token.

  • key (bytes) – The private key used to create and verify the token.

  • authzid (Optional[str]) – Limits the token to authorizing as this identity.

  • location (Optional[str]) – Arbitrary metadata string, application-specific.

  • expiration (Optional[datetime]) – An expiration when this token shall no longer be valid.

Return type:

str | None

class pymap.token.macaroon.MacaroonCredentials(identity, serialized, admin_keys)[source]¶

Authenticate using a Macaroon token. Tokens may be created with either get_login_token() or get_admin_token().

Either type of token may use a caveat with time < and a fromisoformat() string to limit how long the token is valid.

Parameters:

  • identity (str) – The identity to be authorized.

  • serialized (str) – The serialized macaroon string.

  • admin_keys (Set[bytes]) – The admin token string.

property identifier: str¶

Any string that can facilitate the lookup of the token key.

property role: str | None¶

A specialized role granted by the token.

property authcid: str¶

The authentication identity, e.g. a login username.

property authzid: str¶

The authorization identity. The authcid identity must have sufficient privileges to assume this identity for the authentication attempt to succeed.
