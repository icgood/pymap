property backend_capability: BackendCapability¶

Allows backends to declare support for IMAP extensions and other capabilities.

property users_file: str¶

Used by the default find_user() implementation to retrieve the users file path from the command-line arguments. The users file is given as the first positional argument on the command-line.

This file contains a valid login on each line, which are split into three parts by colon (:) characters: the user name, the mailbox path, and the password.

The password may contain colon characters. The mailbox path may be empty, relative, or absolute. If it is empty, the user ID is used as a relative path.

property base_dir: str¶

The base directory for all relative mailbox paths. The default is the directory containing the users file.

pymap.backend.maildir.flags¶

class pymap.backend.maildir.flags.MaildirFlags(keywords)[source]¶

Maintains a set of IMAP keywords (non-standard flags) that are available for use on messages. This uses a custom file format to define keywords, which might look like this:

0 $Junk
1 $NonJunk

The lower-case letter codes that correspond to each keyword start with 'a' for 0, 'b' for 1, etc. and up to 26 are supported.

Parameters:

keywords (Sequence[Flag]) – The list of keywords available for use on messages.

See also
